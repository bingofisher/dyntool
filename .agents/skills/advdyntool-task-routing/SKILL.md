# AdvDynTool 任务分级与角色路由

在 AdvDynTool 仓库中，先判断任务复杂度，再决定是否只由主代理完成、是否优先组合多个 skill、以及是否必须先询问用户启用子代理。

## 规则
- 简单任务：默认只由主代理完成，不启用子代理。
- 稍微复杂任务：默认仍由主代理完成，优先组合多个适用 skill；如果只有一个 skill 真正适用，则只用一个。
- 十分复杂任务：只有在整个仓库重构、跨层联动、公开 API / 单位 / 存储 / 默认行为变化等情形下，才考虑启用子代理。

## 强制询问
- 只要判断“有必要使用子代理”，就必须先询问用户是否启用。
- 例外只有两种：
  - 用户已经明确指定要用子代理。
  - 用户明确在询问是否应该使用子代理或如何使用子代理。

## 输出
- 明确给出任务分级。
- 明确给出建议启用的角色，而不是笼统说“开几个子代理”。
- 明确指出哪些改动必须串行，哪些可以并行。
