# GUI 前端架构与路线
稳定性：`Internal API`

本页用于固定 `src/dyntool_gui/` 当前有效的前端标准。若与 `docs/superpowers/specs/2026-04-24-*` 系列规格冲突，以 `2026-04-24` 规格为准；本页的职责是把这些规格收束为开发者日常可直接引用的统一口径，而不是保留历史探索稿。

## 当前标准

- GUI 继续采用 `PySide6 Widgets + QMainWindow + QDockWidget + QStackedWidget`
- 不修改 `dyntool` Public API
- 当前主导航固定为 `总览 / 导入与筛选 / 数据处理 / 图形绘制`
- `SUBSET / EXPORT` 不再作为主导航页面，只允许作为内部兼容别名或迁移中的状态对象存在
- 视觉 token 与 Qt/Matplotlib 主题 hook 统一由 `ThemeManager + ThemeTokens` 管理
- 4 页正式工作台统一使用页面头部组件，固定“标题 + 副标题 + 当前摘要”语气
- 底部任务面板统一承载任务、日志、问题、导出、审查等详细状态
- 中央工作区优先；右侧 facts panel 不再作为正式结构
- `2560x1440` 等 2K 横屏场景采用横向工作台密度：窄对象树、薄任务区、紧凑导航、低高度页头、中心表格/画布优先

## 全局壳层

当前有效的窗口壳层固定为三层：

1. `项目上下文条`
2. `主导航条`
3. `当前页工具栏`

正式页面头部固定为：

- 页面标题
- 页面副标题
- 当前摘要

页面头部只承载“本页职责和当前状态”，不承载底部任务面板的详细日志、错误清单和长任务记录。

## 2K 横屏工作台合同

当可用屏幕宽度不小于 `2200` 且高度不小于 `1200`，或窗口宽高比不小于 `1.65` 时，GUI 进入横屏工作台布局口径。该口径只调整 `src/dyntool_gui/` 内部展示职责，不改变 `dyntool` Public API、存储格式、单位语义或 4 页主导航。

- 左侧对象树是窄导航栏，默认目标宽度为 `170-180px`，最大宽度不超过 `220px`
- 底部任务区是薄监控区，默认目标高度为 `96-112px`，详细日志由用户手动展开
- 顶部 4 页导航按钮高度不超过 `30px`
- 统一页面头部使用横向单行摘要栏，高度不超过 `64px`
- 导入与筛选页的上方导入链路是横向控制带，高度不超过 `240px`
- 数据处理页和图形绘制页的左侧配置栏最大宽度不超过 `320px`
- 图形页底部结果页签高度不超过 `136px`，不得挤压中央画布
- 页面空态只允许短状态句，不允许把“后续轮次接入”“占位”等历史文案作为正式用户主流程内容

主工作区固定为：

- 左侧：全高对象树
- 中央：当前页主体工作区
- 底部：默认折叠的任务面板

禁止继续使用下列历史口径：

- 6 页主导航
- 独立“交付”主页面
- 独立“筛选与子集”主页面
- 右侧 facts panel 作为正式布局组成
- 在页面主结果区堆放耗时、最近动作、推荐下一步、输出目录等任务摘要

## 页面职责

### 总览

- 只展示项目上下文、当前主样本集摘要、当前范围摘要、最近状态摘要
- 默认隐藏对象树
- 不承担复杂参数编辑、结构化筛选、处理执行或绘图配置

### 导入与筛选

- 承担样本/样本集接入、结构化检查、字段发现、筛选规则构建、数据预览、子集创建与集合组合
- 这是当前唯一承载“筛选与子集”语义的正式页面
- 高级参数必须内嵌在接入参数组内折叠，不允许漂浮到页面其他区块
- 排序、分页、列过滤属于数据预览状态，不属于可持久化的筛选规则

### 数据处理

- 只负责处理与评价，不混入导入与筛选语义
- 优先消费全局 `current_scope`
- 主区负责方法选择、参数编辑和结果浏览
- 任务进度、错误详情、批处理统计统一进入底部任务面板

### 图形绘制

- 只负责 `PlotDataset -> PlotTheme -> plotter` 主链
- 页面主体固定为 `左侧配置 + 中央图形预览`
- 右侧 facts panel 已从正式结构中移除
- 图形页只暴露 plotting 正式 schema 口径，不把运行时字段冒充 TOML schema

## 对象树与状态边界

对象树当前标准结构以 `2026-04-24` 规格为准，开发口径固定为：

- `当前主样本集`
- `子集集合`
- `其他集合`
- `图形记录`
- `导出记录`

`ProjectSession` 是唯一会话状态源，负责保存：

- 项目上下文
- 当前主样本集引用
- 当前范围 `current_scope`
- 导入状态
- 筛选/子集状态
- 处理状态
- 绘图状态
- 底部任务面板所需摘要

页面组件只负责收集输入、发起请求、显示摘要，不直接拼接底层业务调用细节。

## 实现分层

| 层 | 责任 | 说明 |
|---|---|---|
| `MainWindow` | 工作台外壳 | 负责三层壳层、Dock、菜单与全局动作 |
| `ProjectSession` | 唯一状态源 | 管理跨页状态、当前范围与各模块摘要 |
| `Manager` | 动作执行 | `ImportManager / ProcessingManager / PlotManager / ExportManager` 负责后台任务与状态写回 |
| `Workspace Widget` | 页面交互 | 负责参数输入、请求发射、主区摘要与结果展示 |
| `ThemeManager` | 唯一视觉出口 | 统一 Qt Widgets 与 Matplotlib 的主题 token 和 hook |

## 当前需要明确避免的误读

- 不能再用“文档里有 6 页，所以代码应该做 6 页导航”来评审当前 GUI
- 不能把 `SUBSET / EXPORT` 是否仍存在内部状态对象，误判为“必须恢复成独立页面”
- 不能把历史设计稿里的右侧信息栏，当成当前可接受的正式结构

## 迭代路线

当前迭代顺序以 `2026-04-24-gui-iteration-plan-by-sequence.md` 为准，开发者摘要如下：

1. 清理旧 `SUBSET / EXPORT` 导航残留、旧动作别名和重复 handler
2. 固定全局三层壳层与 Dock 行为
3. 重构对象树与当前范围模型
4. 重构导入与筛选页的结构化合同
5. 收口数据处理页
6. 收口图形绘制页
7. 统一底部任务面板、文案、验证与回归
