# GUI 按钮矩阵
稳定性：`Internal API`

本页用于固定当前 4 页 GUI 标准下的按钮、菜单动作与二级路由合同。评审“按钮是否有规划”“跳转是否明确”时，应优先对照本页，而不是从 `_trigger_action()`、页面控件连接或历史设计稿中反推。

## 适用范围

- 当前正式主导航固定为 `总览 / 导入与筛选 / 数据处理 / 图形绘制`
- `SUBSET / EXPORT` 不是独立主页面，只作为内部状态或工作区能力存在
- 图形页正式结构仍为 `左侧配置 + 中央视图预览/结果`，不恢复右侧 facts panel
- 本页只覆盖 `src/dyntool_gui/` 当前用户可见按钮与菜单动作，不新增 `dyntool` Public API

## 六类动作

| 动作类型 | 定义 | 允许改写会话状态 |
|---|---|---|
| 页面切换 | 只切换工作台页面或焦点区域，不直接执行业务 | 否 |
| 主链执行 | 直接调用 `MainWindow -> Manager` 主链执行业务动作 | 是 |
| 预览查看 | 打开只读预览容器或刷新预览结果，不写入核心业务结果 | 仅预览摘要 |
| 预检校验 | 检查前置条件、展示缺项与待补算动作，不直接完成最终业务 | 仅校验摘要 |
| 补算引导 | 直接触发 `compute_required`，或把用户送到对应页面继续补算 | 可能 |
| 信息查看 | 打开只读表格、列表或说明容器，不伪装成可执行操作 | 否 |

## 全局规则

- 所有正式主流程按钮都必须落入上述六类之一
- 不允许出现“按钮看起来可执行，但实际只弹占位说明”的主流程入口
- 页面切换失败视为实现错误；主链执行失败统一使用中文错误提示，并把细节写入底部任务/日志/问题面板
- 预览查看与预检校验在无数据时显示空态，不再使用“后续轮次接入”“占位”文案
- `_trigger_action()` 只承担二级路由，不承载新的业务规则本体

## `_trigger_action()` 二级路由

`MainWindow._trigger_action()` 当前固定承担以下职责：

1. 总览快捷动作别名映射：`接入主样本集 / 管理子样本集 / 开始分析 / 快速出图 / 去交付`
2. 信息查看路由：`查看 metadata 字段`、`查看支持 categories`
3. 预览与预检路由：`导入预览`、`查看结果预览`、`打开大图预览`、`交付预检`、`代码审查结果`
4. 跨页动作路由：`构建子集`、`执行工程导出`
5. 明确保留为只读说明的内部动作：`设置主集`、`设置对比集`

禁止在 `_trigger_action()` 中继续堆叠新的业务规则本体；真正的处理、绘图、导出执行逻辑必须继续下沉到现有主链方法与 manager。

## 失败反馈合同

- 页面切换：原则上不弹业务错误；若目标页面不存在，视为实现缺陷
- 主链执行：统一中文错误弹窗，并把详细状态写入底部任务/日志/问题面板
- 预览查看：无数据时显示空态容器，不显示“后续轮次接入”
- 预检校验：必须同时展示当前缺项、待补算动作和目标输出/目标页面
- 补算引导：上下文足够时直接发出 `compute_required`；上下文不足时先切页或保留当前范围

## 测试要求

- 六类动作至少各有一条 GUI 测试
- 总览页 5 个快捷按钮必须有明确去向断言
- `_trigger_action()` 不得把主流程入口回落到 `PlaceholderDialog`
- 导入预览、结果预览、大图预览、导出预检在无数据时必须显示空态而不是占位说明
